    1. What is Semantic HTML?

	tag that describes what it does, like < table > .
 
    2. What is HTML used for? 

	send commands to a web browser to tell the browser what to display and how.

    3. What is an attribute and where do we put it? 

	provides more specific information about an element, and is located within < > after the tag.

    4. What is the h1 tag used for? How many times should I use it on a page?

	main header, should be used only once per page.

    5. Name two tags that have required attributes

	<a> and <img>

    6. What do we put in the head of our HTML document? 

	title, links to css and js files

    7. What is an id?

	a CSS for only one element.

    8. What elements can I add an id to? 

	any element for which you want a unique set of style commands.

    9. How many times can I use the same id on a page?

	it *can* be used many times, but you should try to keep an id for one specific element;  classes and other CSS elements are better for styles that affect multiple HTML elements.
 
    10. What is a class? 

	a CSS for several similar elements.

    11. What elements can I add a class to? 

	any elements that you want to style similarly to other elements.

    12. How many times can I use the same class on a page? 

	as many as you'd like, as long as the code works!	

    13. How do I get my link to open in a new tab?

Stretch Questions

    1. What is the difference between an inline element and a block element?

	blocks are bullies that push around content in the display, while inlines are ninjas that give content its aspects without necessarily moving other content around the display.

    2. What happens when an element is positioned absolutely? 

	its position is determined by the position of its parent element

    3. How do I make an element take up only the amount of space it needs but also have the ability to give it a width? 

	set the width and let the height vary proportionally with the width.

    4. Name 3 elements that are diplay block by default, 2 elements that are display inline by default and 1 element that is display inline-block by default

	block . . . header, footer, section
	inline . . . a, span
	inline-block . . . img

    5. In your own words, explain the box model. What is the "fix" for the box model, in other words, how do we make all elements respect the width we've given them? 

	box model refers to the fact that every element can have a margin, border and padding around the content, as if it were packaged in a box.  The "display: flex" property value will allow elements to follow the best fit for the available width.
